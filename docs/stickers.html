Sticker pack JSON — spec

How sticker packs are structured, how to create them and how to use the Sticker Pack Creation tool.

Where the tool lives

The Sticker Pack Creation tool is at: /utils/stickerpacks.html. Use it to author, export and import .dpfstickers files.

File format

Sticker packs exported by the tool are JSON files with the extension .dpfstickers. These can be imported for use in the app, or uploaded directly to the store.

{
  "packId": "my-pack-id",
  "name": "My Pack Name",
  "description": "Short description (optional)",
  "author": "author@example.com",
  "date": "2024-01-01T12:00:00.000Z",
  "stickers": [
    {
      "id": "sticker1",
      "name": "Cool Sticker",
      "resource": "https://example.com/image1.png"
    },
    {
      "id": "sticker2",
      "name": "Another",
      "resource": "https://example.com/image2.png"
    }
  ]
}

Field notes:

  • packId — unique id for the pack (used when importing or selling).
  • name — display name of the pack.
  • description — optional description.
  • author — contact / author identifier.
  • date — ISO timestamp when exported (optional but recommended).
  • stickers — array of sticker objects.
  • Each sticker object must contain id, name and resource (URL/local resource).

Creating stickers

Use the tool at /utils/stickerpacks.html to add images, assign IDs and names, and export the JSON file. Important rules:

  • IDs must be unique within a pack.
  • Images must be accessible via HTTPS and ideally 200–250px square for best results.
  • Include at least 2 stickers per pack.

Distribution & store

Sticker packs can be:

  • Imported directly into the app via your file manager.
  • Uploaded to the official store for sale or discovery (packs can be monetized).

Importing into chats

Packs uploaded to the store or saved locally can be imported for use in chats. When users install the pack, stickers are available in the chat picker (depends on app version and settings).

Tips & best practices

  • Prefer PNG with transparency for stickers.
  • Host images on a reliable HTTPS CDN if possible.
  • Keep sticker IDs short and descriptive (no spaces).
  • Test import/export using the tool before publishing.
