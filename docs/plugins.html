Plugins — Distribution & Structure

How plugins must be packaged, localized and published to the official store.

Plugins must be distributed only through the official DarkPurpleOF's Website Development Items Store. Read the rules below carefully — plugins that don't follow the required structure or distribution steps will be rejected.

Plugin structure

The plugin root must contain the following layout:


/plugin
  screens/
  manifest.ini
  icon.png
  extra.toml
  data/

Notes:

  • /screens/ — place UI screens (HTML) here.
  • manifest.ini — required manifest with plugin metadata (example below).
  • icon.png — plugin icon (recommended 128×128 or 256×256 PNG).
  • extra.toml — optional extra config data (TOML format).
  • /data/ — optional storage for plugin resources (images, JSON, etc.).

Localization / language files

Plugin manifests, icons and screen files can use language suffixes to provide localized versions. Append one of these suffixes to the end of the filename:

-es  (Spanish)
-ru  (Russian)
-ja  (Japanese)
-de  (German)

Examples:

  • manifest-de.ini — loads the manifest in German.
  • icon-es.png — Spanish icon variant.
  • screens/credits-ru.html — Russian credits screen.

The manager resolves language-specific files by suffix; if a localized file isn't found it falls back to the base filename.

Manifest format

The manifest is a simple INI-style file with the following fields. All fields are plain text (no quoting needed):


id=idexample123
name=Example Plugin
author=email@email.com
version=1
description=Hello DarkPurpleOF's Website
license=
requires=dependency1[1.1],dependency2[*]
tags=tag1,tag2,tag3

Field notes:

  • id — unique plugin id (used by the manager/store).
  • name — display name.
  • author — contact email (used for verification).
  • version — numeric or semver-like value.
  • description — short description shown in the store.
  • license — optional license identifier or text.
  • requires — comma-separated dependencies with optional version constraints.
  • tags — comma-separated tags for search/filtering.

Screens & root pages

Place UI pages under /screens for in-manager display. You may also add special root pages inside the plugin root for extra behavior:

  • /about.html — optional credits/about page.
  • /settings.html — optional settings page (opened from plugin manager).
  • /index.html — if present, an Open button will appear when the plugin is installed. Index is only used at installation time; the manager will not open index.html after install.
  • /donations.html — optional donations page if you want to ask for donations.

Publishing & store rules

Before publishing, compress your plugin folder into a ZIP file and then rename its extension to .dpplugin. Upload the resulting .dpplugin file to the official verified plugin site using your DarkPurpleOF account.

Important steps:

  1. Create a ZIP of the plugin root (the ZIP's root must contain the files shown in the structure section).
  2. Rename your-plugin.zip → your-plugin.dpplugin.
  3. Sign in to your DarkPurpleOF's Website account and upload the .dpplugin to the verified plugin store.

If you do not have a DarkPurpleOF's Website account, you must create one to upload plugins. If you do not have permission to upload development items, contact support for a check and permission:

Final notes

  • Only plugins distributed through the official store will be listed and discoverable.
  • Use language suffixes (-es, -ru, -ja, -de) to provide localized manifests, screens and icons.
  • Make sure your manifest id is globally unique and your author contact is valid.
